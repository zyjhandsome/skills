# 需求与代码事实简报模板

用于标准 / 高风险车道。快速车道使用 `SKILL.md` 中的轻量任务契约。

写入活跃 OpenSpec change 的 `proposal.md`（或等价 why/what 槽位）；这是简报唯一的物理载体，不得另建 `brief.md`。

文档分三块，用**标题名**定位（不要用章节号）：

1. **意图**
2. **代码事实**（含证据表）
3. **消歧与闸门**

产品决策只写入「开放问题清单」；闸门批准只写入「状态源与工件位置」。

## 验收（标准 / 高风险简报）

- 「意图」：目标、成功标准、边界/非目标已写明
- 「挂载点候选」：至少有 1 个真实存在的必选路径
- 「波及线索」：有具体波及预判（非空话）
- 「证据表」：「代码事实」关键结论有对应事实/推断行（或节内已绑证据）
- 「开放问题清单」：已对用户展示；阻塞项不得在未决定时标为 `decided`；产品决策只落在此表
- 「澄清完整性扫描」：已覆盖实际适用维度；缺失事实已定点调查；新增阻塞决策已进入「开放问题清单」并处理
- 「风险定级与闸门建议」：写清命中 / 未命中 / 不确定 / 闸门建议
- 「Explore 交接消费」：有 explore handoff 时五条均已勾选；无则整节为 `N/A — 无 explore handoff`；仅聊天摘要或空节 = 验收失败
- 「状态源与工件位置」：指向唯一状态源；批准记录只落在此处，无并行竞争副本
