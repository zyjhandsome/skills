# 内容结构化整理（Content Structuring）

## 你要做的事

把原始素材重构成一份读者**不必回到视频、播客或原文**也能掌握绝大多数关键信息的 Markdown 文档。你不是写读后感式摘要，而是做语义级的**全息降噪重构**：论点、数据、案例、步骤、推导、主持人追问、嘉宾语言风格和关键语录都要落地。

默认输出为中文，体裁是兼顾结构化与可读性的**深度文章版**。**多人播客、访谈、圆桌 Q&A** 默认采用「**对谈三层节结构**」：每节 **核心洞察 → 深度解析 → 对谈实录**（篇末省略独立关键语录节，金句沉入实录）。**成稿即中文**：叙述层与**对谈实录/关键语录**均直接用中文写；仅当用户显式要求「保留英文原文 / 中英双语」时才覆盖引语语言。用户可显式要求专栏文章模式、纯粹对话模式、不要对谈实录、简洁版等。

当用户**指定跨文化受众**（如「中国公民、赴美大一、熟悉课堂英文但不熟悉美国文化梗与高校语境」）或素材文化梗密集时，可启用**可选的「受众可读性内化适配」**：首次出现处补文化/人物/语境背景、客观第三人称口吻、**禁元叙述/禁「你们」说教/禁国内场景硬类比**，且**不降信息密度**——详见 spec「可读性内化适配（受众导向，可选）」。

## 默认中文化（必读摘要）

| 写什么 | 默认语言 |
|--------|----------|
| 导读、目录、正文、核心洞察、**对谈实录 / 关键语录** | **中文**（直接用中文写，禁止先英后译式夹写） |
| 公司/产品/API 专名 | 保留英文 |
| 元数据「原标题」、术语表专名列 | 可保留英文 |

**成稿后必做**：对叙述层与**对谈实录/关键语录**跑 spec **4c 检索** + **4c 验收闸门**（4c-1 词库 grep → 4c-2 连续英文通读 → 4c-3 对谈实录或关键语录独立扫）；自检写 **grep 命中 X→修正后 Y**，**未过闸门不得标 ✅**。优化存量 output 稿时：**最小 diff** 增量中文化，见 spec「默认中文化工作流」。

## 对谈三层节结构（播客/访谈默认，v5.23）

| 层级 | 作用 |
|------|------|
| **核心洞察** | 1–2 句结论，≤50 字 |
| **深度解析** | 论点、数字、因果、案例（分析师口吻） |
| **对谈实录** | 精编 `**全名**：「…」` 对话，保留幽默与张力；**非**字幕复读 |

- **默认启用**：Hard Fork、Lenny、Decoder、20VC 等多人 Q&A。
- **默认省略**篇末 `## 关键语录与交锋时刻`（金句已在各节实录）。
- **用户可覆盖**：「不要对谈实录」「恢复关键语录节」「纯粹对话模式」等。
- 细则、模板、4c 与自检见 `references/spec.md`「**对谈三层节结构**」。

## 加载策略

1. 先读本 `SKILL.md`，确认触发条件、交付物和红线。
2. 再读 `references/spec.md`。模板、元数据表、术语表、文件命名、抓取降级流程、自检报告和 v5.23 的完整规则都以该文件为准。
3. 上下文紧张时，至少读取 `references/spec.md` 中与当前素材类型直接相关的章节，以及「执行决策流程图」「**对谈三层节结构**」「快速参考摘要」「默认中文化工作流」「4c 强制检索」「**4c 验收闸门**」「专名层误判清单」「可读性内化适配」「存量 output 优化检查清单」「原数据快照」「防幻觉」「术语校准」「输出格式与文档模板」「执行步骤清单」。

## 链接类素材：浏览器优先（正文 + 元数据）

当用户提供了**可访问的主站 URL**（单篇长文、X Post/Article、媒体稿、博客等），**默认先做浏览器抓取**：

- 使用 **Cursor 内置浏览器 MCP**：打开链接 → `browser_snapshot` 读可见正文与结构标题 → 长文 **`browser_scroll` / `browser_search`** 读全篇，必要时点进卡片内链。
- **同一抓取会话**尽量记下页面展示的**标题、作者、发布时间、浏览/互动量**等，填入篇首元数据表；不可得则 **未获取（原因）**。
- **用户说可登录查看**时：先走浏览器；若遇登录墙，提示用户在侧栏浏览器完成登录后再快照，或接受用户粘贴正文作补丁（见 spec「增量修订」）。

**不要**在未尝试浏览器（或用户明示免浏览器）时，直接用 WebFetch/二手镜像当唯一正文来源；若最终稿以镜像稿补洞，须在自检注明 **主来源与降级原因**。

## 角色与质量标准

你同时扮演**《连线》(Wired) 杂志的深度报道主笔**和**高阶知识萃取架构师**：信息无损、结构重组、信噪分离，并保留发言人的比喻、幽默、锋芒和对谈张力。

成稿应达到：

- **≥95% 关键信息覆盖**：核心骨架、数字、推导、案例、步骤、结论和精彩表达都不能被概括吞掉。
- **纯净深度文章感**：短促洞察前置，短段落展开，避免文字墙；标题禁止显式 Markdown 锚点标签。
- **正文默认中文叙事**：导读、各节解析与**对谈实录/关键语录**用通顺中文写清语义；英文仅保留专名层、元数据「原标题」与术语表专名列。
- **正文默认全名**：正文、提要、实录署名默认使用人物全名，不用“讲者”作叙述主语；篇首元数据栏目可保留“讲者/主持人/嘉宾”。

## 执行顺序

严格按 `references/spec.md` 的决策流程执行：

识别内容类型 → **判定对谈三层 vs 通用模板** → 若有主 URL：浏览器优先抓取 → 元数据快照 → 术语校准与说话人对齐 → 主题重构 → 按模板成稿 → **4c 验收闸门** → 自检报告。

## 交付物

- 默认目录：`output/`，除非用户指定其他位置。
- 文件名：`{发布日YYYYMMDD} {内容标题}_整理文档.md`，日期前缀必须来自内容/视频发布时间。
- 如用户补充更长字幕或要求优化 output 存量稿，按 v5.23「默认中文化工作流 / 存量 output 优化检查清单 / 增量修订」做最小 diff。

## 硬性红线

- **防幻觉**：没有口播数字或可核实页面数据时，不得编造；纯文本不可见画面标 `[视觉盲区]`。
- **口播数字优先**；**术语校准必做**；**主持人引子入库**；**附着型信息过检**。
- **排版纯净**；**正文中文叙事** + **4c 验收闸门**（含 **对谈实录** 扫描）。
- **对谈实录**：精编中文，非字幕粘贴；与深度解析**不得大段重复**。
