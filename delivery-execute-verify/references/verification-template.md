# 验证报告模板

```markdown
# <变更名称>：验证报告

## 范围与状态
- 状态源：
- 风险/闸门：
- 提交/差异：

## 运行与静态证据
| 时间 | 命令/动作 | 退出码/结果 | 失败数 | 覆盖范围 |
|---|---|---|---|---|

### 主验证证据（机器锚点，标签稳定勿改）
（供 `scripts/validate_delivery_change.py --claim-verified` 读取；至少填主验证命令一条。标签「命令 / 时间 / 结果」是校验脚本锚点，改词会导致静默漏检。）
- 命令：<主验证命令或动作>
- 时间：<RFC3339，如 2026-07-16T09:00:00Z>
- 结果：<pass|fail 或退出码，如 exit 0 / 通过>

## 需求验证
| 需求/场景 | 实现证据 | 验证方式 | 结果 |
|---|---|---|---|

## 规格一致性
- 工具/审查：
- 完整性：
- 正确性：
- 一致性：

## 代码审查
### 阻塞项
### 警告项
### 建议项

## 降级项与残余风险
- 跳过/降级检查：
- 批准/原因：
- 覆盖缺口：

## 最终闸门
- 运行/静态检查：通过 / 失败
- 规格核对：通过 / 失败
- 代码审查：通过 / 失败
- 是否达到已验证：是 / 否
- OpenSpec 归档：deferred_to_openspec（本技能不执行）/ not_applicable / blocked(<原因>)

## 资产回写
- 已更新：
- 无需回写，原因：
```

规则补充：

- 当「是否达到已验证」为「是」时，本技能结束于已验证交接；Agent MUST NOT 在本技能内同步 delta 到 `openspec/specs/` 或移动 change 到 archive。
- 下一步应由用户或 Agent 调用已解析的 OpenSpec `archive_change` 操作，保留当前版本的 sync 摘要与确认交互。
- 非阻塞残余（如发布前可选手册冒烟）写入「降级项与残余风险」，不得单独用来阻止已验证收尾。
- git commit / push / PR 仍须用户显式授权，与 OpenSpec 归档分离。

不得根据旧日志、部分命令、未经诊断的重试结果或 Agent 自述标记为“通过”。
