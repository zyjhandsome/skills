# 实施循环模板

## 单任务记录

```markdown
## 任务 <编号>：<可观察行为>
- 状态：待处理 / 进行中 / 被阻塞 / 已实现 / 已验证
- 对应需求/场景：
- 允许范围：
- 禁止范围：
- 失败测试命令/结果：
- 通过测试命令/结果：
- 重构后验证：
- 差异/影响面检查：
- 阻塞项/回流位置：
- 对齐回流摘要（发现、证据、失效工件、所需决定、恢复点）：
```

## 调试记录

```markdown
## 失败现象
- 症状：
- 复现方式：
- 环境/输入：
- 日志：
- 失败路径：

## 根因假设
- 单一根因假设：
- 支持证据：
- 可证伪证据：

## 最小实验
- 最小修改/动作：
- 结果：

## 修复与回归
- 根因修复：
- 原始复现结果：
- 相关回归：
- 对完整验证的影响：
```

## 状态规则

集成失败会阻止整体完成。保留已独立证明的任务状态，只重新打开验收或实现已失效的任务。

## TDD 时序护栏

同一行为切片内：

1. **仅**提交失败测试（或最小断言）并运行 → 必须观测到 RED。
2. **禁止**把「写失败测试」与「写生产实现」放在同一轮并行 tool 调用中。
3. 观测 RED 后，再提交最小实现并运行 → GREEN。
4. 若未观测到独立 RED 就已绿，记为 TDD 时序违规，必要时删掉未验证实现重来（不删用户原有代码）。
