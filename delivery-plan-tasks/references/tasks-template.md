# 任务清单模板

```markdown
# <变更名称>：实施任务清单

## 执行规则
- 权威状态源：
- 风险/闸门：
- 禁止范围：
- 必须执行的最终验证：

## 任务

- [ ] 任务 1：<可观察的纵向行为>
  - 对应需求/场景：
  - 前置依赖：
  - 目标文件/符号：
  - 允许修改：
  - 禁止修改：
  - 实施步骤：
  - 失败测试或已批准替代验证：
  - 验证命令/动作：
  - 预期结果：
  - 迁移/回滚：
  - 完成定义：
  - 负责人/冲突说明：

## 集成顺序

## 最终验证
| 命令/动作 | 覆盖范围 | 预期结果 |
|---|---|---|
```

若任务包含猜测路径、抽象动作、延后测试，或多人所有权重叠却没有集成计划，则不得通过。

> **机器锚点（标签稳定勿改）：** 每条任务的「目标文件/符号」「验证命令/动作」「预期结果」三个标签被 `../../delivery-execute-verify/scripts/validate_delivery_change.mjs` 按字面匹配（支持 ASCII `:` 与全角 `：`）。改动这些标签措辞会导致结构校验静默漏检；如需改名，同步更新该脚本的 `FIELD_PATTERNS`。
